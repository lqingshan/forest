# 需求文档

本目录用于管理 Attendance app 的产品需求材料。

需求材料允许保留 Word、PPT、PDF、Excel 等原始格式，但开发和测试不直接依赖散落附件，而是依赖已确认的需求基线。

## 目录结构

| 目录 | 作用 | 维护规则 |
|---|---|---|
| `source/` | 原始需求材料 | 保留业务方原貌，不轻易改动 |
| `extracted/` | 从原始材料提炼出的 Markdown 摘要 | 用于梳理流程、页面、角色、验收点 |
| `accepted/` | 已确认的需求基线 | 开发、测试、排期以这里为准 |

## 状态约定

| 状态 | 含义 |
|---|---|
| `SOURCE` | 原始材料已归档，尚未提炼 |
| `EXTRACTED` | 已完成结构化提炼，待确认 |
| `ACCEPTED` | 已确认，可作为开发依据 |
| `SUPERSEDED` | 已被新版本替代，仅保留历史记录 |
