# Recharge 文档地图

Recharge 是充值套餐、充值订单和支付联动领域模块。

## 模块定位

`business/domains/recharge` 负责“用户购买积分或套餐”的业务订单，不直接处理支付渠道底层回调，也不直接展示小程序支付页面。

## 需求范围

### 本期范围

| 范围 | 说明 |
|---|---|
| 充值套餐 | 套餐配置、展示和上下架 |
| 充值订单 | 创建订单、订单状态和业务幂等 |
| 支付联动 | 创建支付单并在支付成功后发放积分 |
| 前端模块 | 小程序充值入口和套餐展示能力 |

### 暂不包含

| 暂不包含 | 说明 |
|---|---|
| 支付渠道实现 | 属于 payment domain |
| 积分账户内部模型 | 属于 point domain |
| 复杂营销活动 | 优惠券、促销等后续单独设计 |

## 关键技术点

- recharge 订单是业务订单，payment 支付单是支付渠道抽象。
- 支付成功后需要幂等触发积分发放。
- 充值套餐展示可由 recharge frontend 提供结构和类型，app 小程序负责页面流程和支付调起。

## 职责边界

| 层级 | 职责 |
|---|---|
| recharge domain | 充值套餐、充值订单、支付成功后的业务处理 |
| payment domain | 支付单、渠道参数、支付回调 |
| point domain | 积分余额和流水 |
| app 小程序 | 充值页面、支付调起、结果页跳转 |
