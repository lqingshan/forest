# File 文档地图

File 是文件上传下载、文件元数据和对象存储接入领域模块。

## 模块定位

`business/domains/file` 负责文件资源的业务元数据、上传下载接口和对象存储适配，不承载具体 app 的业务归属规则。

## 需求范围

### 本期范围

| 范围 | 说明 |
|---|---|
| 文件上传 | PC/H5/小程序等端的文件上传入口 |
| 文件下载 | 文件访问、下载和元数据查询 |
| 对象存储 | 接入 starter-object-storage |
| 文件元数据 | 保存文件大小、类型、路径、归属标记等基础信息 |

### 暂不包含

| 暂不包含 | 说明 |
|---|---|
| 业务审批 | 文件是否可用于某个业务流程由业务 domain 决定 |
| 内容审核 | 后续如需要，应单独设计 |
| 大型媒体处理 | 转码、切片、预览等后续再做 |

## 关键技术点

- 文件物理存储由对象存储 starter 适配，file domain 保存业务元数据。
- 上传接口需要校验大小、类型、场景和访问权限。
- 文件归属应表达业务场景，但不应让 file domain 反向依赖具体 app 业务。
- 多文件交互规范参考根 guidelines。

## 职责边界

| 层级 | 职责 |
|---|---|
| file domain | 文件元数据、上传下载接口、业务文件记录 |
| starter-object-storage | OSS/S3/本地存储技术封装 |
| app/domain | 决定文件属于哪个业务对象、是否允许使用 |
| frontend | 上传组件、进度、校验提示和文件选择体验 |
