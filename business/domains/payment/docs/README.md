# Payment 文档地图

Payment 是支付单、支付渠道、支付参数和支付回调领域模块。

## 模块定位

`business/domains/payment` 负责支付抽象和支付渠道接入。它不拥有充值套餐、不直接发放积分、不决定业务订单的最终权益。

## 需求范围

### 本期范围

| 范围 | 说明 |
|---|---|
| 支付单 | 创建、查询和状态流转 |
| 支付渠道 | 微信小程序支付等渠道适配 |
| 支付参数 | 生成前端调起支付所需参数 |
| 支付回调 | 验签、解析、幂等处理和通知业务方 |

### 暂不包含

| 暂不包含 | 说明 |
|---|---|
| 业务权益发放 | 由 recharge、point 等业务模块处理 |
| 订单业务模型 | 支付单不替代充值单或交易单 |
| 多渠道复杂路由 | 后续如需要再设计支付路由策略 |

## 关键技术点

- 支付单表达支付事实，业务订单表达业务权益。
- 支付回调必须幂等，不能重复触发业务发放。
- 渠道参数和证书配置应与环境隔离，不能写入文档明文。
- 小程序支付调起由 app 前端负责，payment 提供参数。

## 职责边界

| 层级 | 职责 |
|---|---|
| payment domain | 支付单、渠道适配、回调处理 |
| recharge domain | 充值订单和权益发放触发 |
| point domain | 积分增加和流水 |
| app frontend | 支付调起和结果展示 |
